Container Queries

Container Queries

  • odklon od @media queries
  • flexibilní layout (grid, minmax())
  • veškerá logika přechází na samotné komponenty
  • komponenta ví, jak se má chovat v různých situacích
  • macro -> micro layouts

Container Queries - použití

  • používá se velice podobně, jako @media queries
  • chrome://flags/ / Container Queries Polyfill
  • co lze měnit
    • layout
    • design
    • content
  • container-name: my-container
```css @container (min-width: 400px) { ... } @container my-container (min-width: 400px) { ... } ```

Cascade layers

Cascade layers

  • větší kontrola specificity selectorů
  • pro @layer platí běžné CSS zákony (poslední vyhrává)
  • ale! to platí v okamžiku prvního výskytu
  • styly mimo @layer mají vyšší speciificitu
  • funguje zanořování @layers
  • vhodné pro zapouzdřování knihoven
  • Chrome developer tools - pěkně zobrazuje
  • už funguje všude

Cascade layers - použití

```css @import url(headings.css) layer(default) @layer default { h1 {...} } /* na začátku stylesheetu */ @layer basic default theme ```

Drobnosti

gap

``` .container-grid { display: grid; gap: 2rem; } .container-flex { display: flex; gap: 2rem; } ```

inset

``` .old { top: 0; right: 0; bottom: 0; left: 0; } .new { inset: 0; } ```

accent-color

``` .root { accent-color: hotpink; } @media(prefers-color-scheme: dark) { .root { accent-color: lightpink; } } ```
  • Example

hwb()

  • hue, whiteness, blackness
  • kromě Opery všude

has()

  • Safari, Chrome
``` .parent:has(.child) { ... } // všechny a, které mají uvnitř img a:has(> img){ ... } // všechny img, které jsou uvnitř figure s figcaption figure:has(figcaption) img { ... } ```
